# Проверка качества распознавания

Сложите тестовые WAV/MP3-файлы рядом с манифестом и укажите для каждого точную
ручную расшифровку ожидаемого финального текста (уже с выполненными командами
пунктуации). В наборе нужны тихая и обычная речь, шум, длинные фразы,
названия и голосовые команды. Не используйте один и тот же голос во всех тестах.

Запуск из корня проекта:

```powershell
.\.venv\Scripts\python.exe .\tools\benchmark_quality.py `
  .\benchmarks\manifest.json --model base
```

Главная метрика — `WER` (доля ошибочных слов, меньше лучше). `CER` помогает
увидеть небольшие ошибки в окончаниях и названиях, а `real_time_factor` показывает
скорость: значение меньше 1 означает обработку быстрее длительности записи.
Отчёт также содержит `by_tag`, поэтому тихую речь, шум, названия и длинные фразы
можно сравнивать отдельно. Для теста тишины оставьте `reference` пустым и добавьте
тег `silence`: поле `hallucinated_words` должно оставаться равным нулю.

Чтобы отдельно измерить пользу словаря, повторите тот же запуск с флагом
`--without-custom-terms` и другим именем выходного файла.

`custom_terms` поддерживает старый список `Codex, PostgreSQL` и явные варианты:
`PostgreSQL = постгрес | пост грес`. Замены выполняются только для целых фраз.

Для настройки детектора речи один и тот же набор можно прогнать с профилями
`balanced`, `sensitive`, `strict` и контрольным `off`:

```powershell
.\.venv\Scripts\python.exe .\tools\benchmark_quality.py `
  .\benchmarks\manifest.json --model base --vad-profile strict `
  --output .\test-output\quality-strict.json
```

Технический выбор профиля доступен только benchmark-инструменту. Обычному
пользователю не нужно разбираться в порогах VAD.

## Запись собственного набора

Список доступных микрофонов:

```powershell
.\.venv\Scripts\python.exe .\tools\record_benchmark_sample.py --list-devices
```

Запись одного примера длительностью восемь секунд:

```powershell
.\.venv\Scripts\python.exe .\tools\record_benchmark_sample.py `
  .\benchmarks\user\manifest.json `
  --id clean-ru-01 `
  --reference "Точная фраза, которую нужно произнести." `
  --seconds 8 `
  --tags real,clean
```

Скрипт показывает текст, запускает обратный отсчёт, записывает выбранный
микрофон, сохраняет моно-WAV 16 кГц и безопасно дописывает manifest. Для тихого
примера добавьте тег `quiet`, для имён — `names`, для терминов — `terms`, для
фонового шума — `noise`. Повтор существующего ID блокируется; осознанная
перезапись выполняется с `--overwrite`.

Каталог `benchmarks/user` исключён из Git: реальные голоса и ручные расшифровки
остаются только на этом компьютере.

## Автоматические шумовые проверки

Воспроизводимый набор из цифровой тишины, слабого комнатного шума, гула и
клавиатурных щелчков создаётся так:

```powershell
.\.venv\Scripts\python.exe .\tools\generate_audio_edge_cases.py
.\.venv\Scripts\python.exe .\tools\benchmark_quality.py `
  .\test-output\vad-edge-cases\manifest.json --model base `
  --fail-on-hallucinations `
  --output .\test-output\quality-vad-edge-balanced.json
```

Для этих примеров `silence_cases` должно быть равно `4`, а
`hallucinated_words` — `0`.

Каждый результат также содержит `speech_seconds` и `speech_ratio`: они показывают,
какую часть файла VAD действительно передал распознаванию. Для обычной речи эта
доля не должна резко падать после изменения профиля, а для тишины должна быть
равна нулю. Опциональный `--max-wer 0.20` делает превышение WER 20% ошибкой
команды и подходит для автоматической проверки перед сборкой.
