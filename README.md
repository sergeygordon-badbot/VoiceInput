# Речка — голосовой ввод для Windows 11

Сайт проекта и бесплатная загрузка: https://ebsf.ru/rechka/

Программа диктовки с автоматическим выбором режима: нажмите глобальную горячую
клавишу, произнесите текст, нажмите её ещё раз — распознанный текст появится в
активном приложении.

Статус проекта: **публичная beta**. Основной сценарий уже пригоден для
ежедневной диктовки, но возможны ошибки распознавания и предупреждение Windows
SmartScreen у неподписанного установщика.

«Речка» использует Whisper/faster-whisper, но не является официальным продуктом
OpenAI.

## Что уже работает

- глобальная клавиша `Ctrl + Пробел`;
- запись с выбранного микрофона;
- трёхсекундная проверка микрофона с оценкой тишины, тихого сигнала и перегруза;
- бесплатное распознавание через публичный Hugging Face Space без регистрации
  и API-ключа — основной режим для слабых ПК и резерв для более быстрых;
- автоматический локальный Whisper: основной режим на достаточно быстром
  компьютере и безлимитный резерв, если публичный сервер недоступен;
- анализ процессора, памяти и GPU перед первой диктовкой с автоматическим
  выбором Tiny или Base и подходящего числа потоков;
- явный выбор распознавания: «Авто», бесплатный «Онлайн» или полностью
  «Локально»; в локальном режиме доступны ручной выбор модели и скорости;
- карточка активного режима, характеристик компьютера и измеренной скорости на
  главной странице;
- предварительный VAD-отсев тишины и шума до запуска тяжёлой расшифровки;
- четыре режима результата: дословно, общение, структурированный промпт для AI
  и свой локальный режим обработки;
- отдельная настройка промпта под ChatGPT, Claude, Gemini или универсальный
  формат и постоянный контекст проекта;
- плавающая плашка с живым индикатором громкости и понятным статусом записи;
- единая финальная расшифровка всей записи без отвлекающего неточного черновика;
- длинная запись через временный локальный файл, который удаляется после
  обработки;
- отзывчивый CPU-режим, оставляющий одно физическое ядро интерфейсу и Windows;
- автоматический выбор CUDA на совместимой NVIDIA-видеокарте с безопасным
  возвратом на CPU;
- режимы декодирования «Быстро», «Баланс» и «Точно»;
- полностью отключаемые мягкие звуки старта и остановки;
- аккуратная пунктуация очевидных вопросов, русское тире и команды «новая
  строка», «новый абзац», «поставь точку», «поставь тире», «поставь дефис»;
- вставка через `Ctrl+V`, прямая Unicode-печать или только копирование;
- восстановление предыдущего содержимого буфера после вставки;
- `Esc` для отмены записи без расшифровки и вставки;
- голосовые команды «отмени», «повтори» и «нажми Enter», срабатывающие только
  когда вся диктовка является командой;
- отдельные вкладки готового текста и исходной расшифровки;
- быстрые фразы, заменяющие короткую голосовую команду на готовый текст, и
  дополнительные профили приложений;
- отключённая по умолчанию локальная история без аудио с переключателем прямо
  на вкладке «История»;
- импорт и экспорт персонализации без истории;
- работа из области уведомлений;
- автоматическая фоновая проверка обновлений и ручная кнопка в настройках;
- проверка SHA-256 установщика перед обновлением;
- смена языка, микрофона и горячей клавиши; в автоматическом режиме локальный
  резерв и скорость подбираются по характеристикам компьютера;
- словарь с точным написанием и явными вариантами распознавания;
- копирование локальной диагностики оборудования без аудио, расшифровки и
  пользовательского словаря;
- необязательный автозапуск вместе с Windows;
- необязательная локальная обработка промптов через Ollama и Qwen3 4B.

В автоматическом сетевом режиме записанная фраза отправляется непосредственно
выбранному публичному Hugging Face Space для распознавания. У «Речки» нет общего
API-ключа и промежуточного сервера. Публичный сервис не гарантирует доступность
или отсутствие лимитов, поэтому Tiny и Base включены в установщик и приложение
автоматически продолжает локально при сетевой ошибке. Во время записи исходный
поток временно хранится в системной папке Windows и удаляется сразу после
остановки или отмены записи. Подробности — в `PRIVACY.md`.
Локальная история включается только вручную и никогда не хранит аудио. В режиме
AI-промпта текст может обрабатываться локальной Ollama; если она не установлена, программа
использует встроенный структурированный шаблон.

## Запуск готовой сборки

1. Откройте папку `dist\Rechka-X.Y.Z\Rechka`, где `X.Y.Z` — версия сборки.
2. Запустите `Rechka.exe`.
3. Дождитесь статуса «Готово к диктовке».
4. Поставьте курсор в нужное поле.
5. Нажмите `Ctrl + Пробел`, говорите и нажмите сочетание ещё раз.

Закрытие окна сворачивает программу в область уведомлений. Полный выход доступен
через значок микрофона рядом с часами.

## Режим вставки

- «Буфер обмена + Ctrl+V» — наиболее совместимый режим. После вставки прежнее
  содержимое буфера восстанавливается.
- «Прямая печать» — не меняет буфер, но отдельные приложения могут блокировать
  программный ввод.
- «Только скопировать» — безопасный запасной режим для нестандартных полей.

Windows запрещает обычным программам ввод в окна, запущенные от администратора.
Если диктовка нужна в административной программе, запустите `Rechka.exe`
с тем же уровнем прав.

## Запуск из исходников

```powershell
.\.venv\Scripts\python.exe main.py
```

Диагностика:

```powershell
.\.venv\Scripts\python.exe main.py --self-test
```

Распознавание файла:

```powershell
.\.venv\Scripts\python.exe main.py --transcribe .\sample.wav
```

Тесты:

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -v
```

Benchmark качества (формат набора описан в `benchmarks\README.md`):

```powershell
.\.venv\Scripts\python.exe .\tools\benchmark_quality.py `
  .\benchmarks\manifest.json --model base
```

Для локальной записи размеченного примера используйте
`tools\record_benchmark_sample.py`; личный набор в `benchmarks\user` не
добавляется в Git.

Сборка:

```powershell
.\build.ps1
```

Установщик Windows:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\build-installer.ps1
```

Скрипт использует официальную portable-версию Inno Setup, проверяет её цифровую
подпись и создаёт единственный установщик
`dist\installer\Rechka-Setup-X.Y.Z.exe`. Установщик:

- не требует прав администратора;
- добавляет программу в меню «Пуск»;
- создаёт рабочий ярлык «Речка»;
- регистрирует штатное удаление через параметры Windows;
- сохраняет пользовательские настройки при удалении;
- удаляет запись автозапуска, если она была включена в программе.

## Обновления

Публичные версии проверяют обновления через GitHub Releases при запуске, каждые
шесть часов во время долгой сессии и повторно через десять минут после сетевой
ошибки. Новая публичная версия обязательна: окно предлагает установить её или
закрыть программу. При скрытом запуске окно выводится на передний план.

Программа принимает только установщик с ожидаемым именем, размещённый на
`github.com`, и перед запуском сверяет размер и SHA-256 из GitHub API.

При запуске из исходников канал обновлений можно задать переменной
`RECHKA_UPDATE_REPOSITORY=OWNER/Rechka` или файлом `release.json` по
образцу `release.example.json`. В релизной сборке этот файл создаётся автоматически.

Чтобы опубликовать новую версию:

1. измените `__version__` в `voice_input/__init__.py`;
2. обновите `CHANGELOG.md`;
3. слейте проверенный pull request в `main`;
4. GitHub Actions самостоятельно создаст тег `vX.Y.Z`, соберёт и опубликует
   установщик, пометит релиз как `Latest` и проверит, что канал обновления уже
   отдаёт новый тег, размер и SHA-256.

Обычный push без нового номера версии не создаёт повторный релиз.
`voice_input/__init__.py` — единственный источник номера текущей версии.
Сборщик приложения, установщик и Windows-метаданные читают его автоматически.
Лендинг получает номер, размер и прямую ссылку на установщик из последнего
GitHub Release при открытии страницы. Не добавляйте текущий номер версии вручную
в README, политику, Inno Setup или HTML лендинга. Номера в `CHANGELOG.md`,
`ROADMAP.md` и тестовых примерах являются историей и не синхронизируются.

Релиз создаётся только после зелёных проверок и успешной сборки. Это защищает
пользователей от незавершённых версий.

Политика конфиденциальности находится в `PRIVACY.md`, правила сообщения об
уязвимостях — в `SECURITY.md`, список сторонних компонентов — в
`THIRD_PARTY_NOTICES.md`.

## Выбор модели

- `Base` — рекомендуемый быстрый режим для обычного процессора; включён в установщик.
- `Small` — точнее `Base`, но заметно медленнее; скачивается отдельно.
- `Medium` — точнее `Small`, но очень медленно работает на CPU; скачивается отдельно
  и занимает около 1,5 ГБ.
- `Turbo` — максимальная точность из доступных вариантов, но на этом CPU очень
  медленная; скачивается отдельно и занимает около 1,6 ГБ.

Рекомендуемый стандарт — `Base` и режим `Баланс`: он немного медленнее, но
аккуратнее быстрого режима. Для минимальной задержки можно вручную выбрать
`Быстро`. Для готового текста используется единая модель и вся запись целиком.

## Режимы результата

- `Дословно` возвращает результат Whisper без AI-переработки. Произнесённые
  команды пунктуации применяются отдельно и могут быть выключены.

- `Общение` сохраняет исходный смысл и формулировку, исправляя пунктуацию,
  слова-повторы, явные звуки колебания и проговорённые самокоррекции. Если
  локальная Ollama доступна, она выполняет эту правку в консервативном режиме;
  без неё действует встроенная безопасная очистка.
- `Промпт для AI` превращает поток мыслей в задачу с целью, контекстом,
  требованиями, ограничениями и ожидаемым результатом.
- `Свой режим` применяет пользовательскую инструкцию через локальную Ollama. Если
  Ollama недоступна, исходный текст не теряется и возвращается с пояснением.

Во втором режиме можно выбрать целевую систему. Для ChatGPT инструкции ставятся
в начало и явно отделяются от контекста; для Claude поддерживается удобное
разделение сложного запроса XML-тегами; для Gemini используется стабильная
структура с отдельно указанными контекстом, задачей и форматом результата.

Для более качественной локальной переработки промптов можно установить Ollama и
загрузить модель `qwen3:4b`. Ollama не входит в установщик «Речки», потому что
её движок и модель занимают несколько гигабайт. Без неё режим остаётся рабочим и
оборачивает исходную речь в понятный универсальный запрос.

## Ограничения публичной beta

- это переключатель «начать/остановить», а не режим удержания клавиши;
- живой текст является предварительным и может изменяться по мере продолжения
  речи; окончательный текст создаётся после остановки;
- на CPU обновление живого текста приходит с задержкой в несколько секунд;
- поля паролей и некоторые защищённые окна могут запрещать вставку;
- первая загрузка дополнительной модели может занимать несколько минут.
